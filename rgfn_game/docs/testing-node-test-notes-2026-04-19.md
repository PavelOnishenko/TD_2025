# RGFN test execution notes (2026-04-19)

## Why `node --test` can fail immediately in `rgfn_game`

Most scenario and system tests import built modules from `rgfn_game/dist/**`. The `dist` folder is generated by TypeScript build and is gitignored.

If `rgfn_game/dist` does not exist, `node --test` reports `ERR_MODULE_NOT_FOUND` for many test files.

## Reliable local sequence

From repository root:

```bash
npm run build:rgfn
cd rgfn_game
node --test
```

This is equivalent in spirit to the root helper script `npm run test:rgfn` (which builds first, then runs the rgfn tests).

## Practical troubleshooting checklist

1. Run `npm run build:rgfn` first.
2. Run `cd rgfn_game && node --test`.
3. If you see path errors to `dist/**`, rebuild and verify `rgfn_game/dist` exists.
4. If UI scenario tests fail, check that test stubs include newly added HUD elements/callbacks used by controllers.
5. If quest/village tests fail after runtime changes, compare expected return object shapes in tests against current implementation signatures.
